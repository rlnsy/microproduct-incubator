## Overview

Transition notebooks, benchmarks, or one-off analyses into reproducible software without hiding assumptions.

## Getting started

- Mirror experimental notebooks with referenced datasets and versions inside `architecture.md`.
- Capture promotion criteria explicitly in `roadmap.md` and `evaluation.md`.

## Repository hygiene

- Commit to honest `maturity` fields in `product.yaml` early.
- Replace `src/` and `tests/` placeholders with runnable pipelines.
- Upgrade `.github/workflows/checks.yml` when evaluation harness runners exist.
