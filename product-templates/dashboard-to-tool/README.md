## Overview

Use this scaffold when KPI walls exist but routed actions—and accountability—do not.

## Getting started

- Map every KPI to a mandated next step documented in `product-brief.md`.
- Explain refresh cadences and alerting owners in `data-contract.md`.

## Repository hygiene

- Treat `evaluation.md` as the guardrail tying UX changes to observable outcomes.
- Replace `src/` and `tests/` placeholders with runnable code.
- Upgrade `.github/workflows/checks.yml` after choosing your UI stack.
